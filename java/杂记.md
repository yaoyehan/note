### Nginx和网关有啥区别？
1. **Nginx 的核心功能**

| 功能     | 说明                       |
| ------ | ------------------------ |
| 反向代理   | 将客户端请求转发到后端服务器           |
| 负载均衡   | 分发请求到多个后端服务器             |
| 静态资源服务 | 直接提供 HTML、CSS、JS、图片等静态文件 |
| 限流     | 基于 IP、连接数等基础限流           |
| URL 重写 | 重写和重定向 URL               |
2. **网关（gatway）**

| 功能   | 说明                    |
| ---- | --------------------- |
| 路由转发 | 根据路径、Header 等路由到不同微服务 |
| 统一认证 | JWT、OAuth2 等认证授权      |
| 服务治理 | 熔断、降级、重试、超时控制         |
| 监控告警 | 请求日志、性能监控、告警          |
总结：
**Nginx**更多的是用于==反向代理==和==负载均衡==，如果用nginx做服务转发，每次新增节点和扩容，都得手动调整配置，而**网关**可以接入注册服务中心，可以==借助注册服务中心自动实现请求转发==，无需手动配置，而且网关可以实现很多==通用==的业务层面的==服务治理==，如==监控、鉴权==等，不用每个服务都实现一次。
